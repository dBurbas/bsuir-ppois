# Лабораторная работа №1

- Вариант: Англо-русский словарь, Множество

## Описание
Выполненная лабораторная работа включает два независимых модуля:
- **Неориентированное канторовское множество** *(с поддержкой вложенных множеств)*\
Реализовано через класс CantorSet с контейнером через std::vector.

- **Англо-русский словарь**\
Реализовано через класс Dictionary с самостоятельно реализованным контейнером\
бинарное дерево поиска *(но в перспективе можно заменить на стандартные контейнеры вроде std::map)*.

## Возможности
- CantorSet:
    Формирование множества из строки, как из char *, так и из std::string;
    Поддерживаются атомарные элементы и вложенные множества;\
    Поддерживаются все необходимые базовые операции теории множеств:
    - проверка на пустое множество;
    - добавление элемента;
    - удаление элемента;
    - определение мощности множества;
    - проверка принадлежности элемента множеству (оператор []);
    - объединение двух множеств (операторы + и +=);
    - пересечение двух множеств (операторы * и *=);
    - разность двух множеств (операторы - и -=);
    - построение булеана данного множества.

- Dictionary: 
    - добавление нового английского слова и перевода для него (оператор += для работы как со
    строками в стиле С - char *, так и в стиле С++ - std::string);
    - удаление существующего английского слова из словаря (оператор -=);
    - поиск перевода английского слова (оператор []);
    - замена перевода английского слова (оператор []);
    - определение количества слов в словаре;
    - возможность загрузки словаря из файла;
    - валидация формата слов с помощью утилит IsEnglishWord/IsRussianWord(английское — только ASCII‑буквы с разрешёнными дефисами/пробелами; русское — не‑ASCII буквы с разрешёнными дефисами/пробелами).​

- Внутреннее хранилище словаря — самостоятельно реализованное обобщённое бинарное дерево поиска с операциями вставки, удаления, поиска, обходом in‑order и сравнением на равенство/неравенство.

## Требования
Рекомендуется компилятор с поддержкой стандарта C++17 и стандартной библиотеки, так как API использует строки, потоки и контейнеры STL в заголовках и реализациях обоих модулей.

## Примеры кода
CantorSet:
```cpp
CantorSet a("{a, b, {c}}");// формирование множества через строки
CantorSet b("{b, d}");     
CantorSet u = a + b;       // объединение
CantorSet i = a * b;       // пересечение
CantorSet d = a - b;       // разность
CantorSet p = a.PowerSet();// булеан
std::cout << u << "\n";
```

Dictionary:
```cpp
Dictionary dict;
dict += "hello:привет"; // добавление слова через строку
dict += std::make_pair(std::string("world"), std::string("мир")); // добавление слова через пару
std::cout << dict["hello"] << "\n"; // поиск и вывод перевода
dict -= "world"; // удаление пары слов
std::cout << dict; // печать всех пар слов
```
